Project Title: Dodo’s Coffee Bar Website

1. Project Purpose
The purpose of this project is to design and develop a multi‑page website for Dodo’s Coffee Bar, a fictional neighborhood café known for its cozy atmosphere, handcrafted beverages, and community‑centered identity. This project demonstrates my ability to apply PMUPM Chapter 3 principles by defining a clear scope, establishing boundaries, and planning a structured web development effort from concept to deployment. The completed website will serve as a portfolio‑ready artifact showcasing both technical and project‑management skills.

2. Project Description
This project involves creating a themed, multi‑page website that represents the brand identity and customer experience of Dodo’s Coffee Bar. The site will highlight the café’s story, menu offerings, ambiance, and values through intentional design and organized content.

The visual aesthetic will use a warm, earthy palette inspired by natural wood, espresso tones, and soft ambient lighting. Typography, layout, and imagery will reinforce the café’s welcoming, community‑oriented personality. The website will be built using HTML and CSS, published through GitHub Pages, and structured to reflect thoughtful planning and consistent branding.

3. Desired Results
A fully functional, multi‑page website representing Dodo’s Coffee Bar

Clear, organized content that communicates the business’s offerings and identity

Functional navigation connecting all pages

9. Loops
This project will follow short, iterative development cycles to maintain alignment with the scope and prevent scope creep.

Loop 1 – Structure & Layout:  
Build the basic HTML structure and navigation for all pages.

Loop 2 – Branding & Styling:  
Apply the color palette, typography, spacing, and overall visual identity.

Loop 3 – Content Development:  
Add written content, menu descriptions, café story, and placeholder images.

Loop 4 – Visual Enhancements:  
Add final images, refine spacing, adjust layout, and ensure brand cohesion.

Loop 5 – Final Review & Deployment:  
Test navigation, validate HTML/CSS, complete final adjustments, and publish to GitHub Pages.

Each loop ends with a brief evaluation to confirm alignment with the approved scope.
