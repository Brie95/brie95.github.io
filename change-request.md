Change Title: Add Embedded Google Map Showing Hometown

1. Proposed Change
The stakeholder is requesting an enhancement to the delivered Midterm Project website: the addition of an embedded Google Map displaying the student’s hometown. This change requires inserting a Google Maps iframe or equivalent embed code into the existing webpage structure. The modification would introduce a new interactive element but does not alter the original project purpose or core deliverables.

2. Reasons for the Change
This request appears to align with several instructional and stakeholder objectives:

Reinforcing course learning: The stakeholder may want students to demonstrate the ability to evaluate and plan for changes after initial delivery, which is a key concept in PMUPM Chapter 6.

Encouraging integration of external resources: Embedding a map requires students to work with third‑party tools, reflecting real‑world project scenarios.

Increasing personalization: The feature adds contextual relevance by connecting the website to the student’s background.

Assessing adaptability: The stakeholder may be evaluating how effectively students apply structured change‑control processes rather than reacting informally to new requests.

3. How This Change Will Add Value
Implementing this change would provide value in several dimensions:

User Experience: An interactive map enhances engagement and makes the site feel more dynamic and informative.

Professionalism: The feature demonstrates the student’s ability to incorporate external content, a common requirement in modern web projects.

Skill Development: Responding to a post‑launch request reinforces the PMUPM principle that projects evolve and require disciplined evaluation of new inputs.

Stakeholder Satisfaction: Addressing the request through a formal change‑control process shows maturity, accountability, and alignment with project‑management best practices.

4. Impact on Project Constraints
Using the PMUPM framework, the following impacts are anticipated:

Time
Low to Moderate Impact.  
Researching embed options, adjusting layout, and testing responsiveness may require 30–60 minutes. Additional time may be needed if styling conflicts arise.

Quality
Potential Positive Impact, with manageable risks.

Risks include layout disruption, inconsistent styling, or reduced accessibility if not implemented carefully.

Budget

Embedding a Google Map is free. The only cost is additional student time, which is considered part of the learning process.
